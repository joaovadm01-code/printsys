# Benchmark PrintSys - Mubisys + Holdprint/Holdworks

## Objetivo

Consolidar o melhor das referencias publicas de Mubisys, Holdprint/Holdworks e do prompt anexado para orientar a proxima repaginacao do PrintSys sem copiar marca, codigo, icones, SVGs, imagens ou propriedade visual fechada.

## Fontes consultadas

- Mubisys: sistema para grafica e comunicacao visual, com fluxo do primeiro contato a entrega, modulo comercial, financeiro, producao, PCP, estoque e app de acompanhamento. Fonte: https://mubisys.com/
- Holdworks: plataforma para comunicacao visual com orcamento tecnico, producao, estoque, financeiro, PCP, MRP, relatórios e previsto x realizado. Fonte: https://www.holdworks.ai/
- Holdprint Desktop: ERP para comunicacao visual com orcamentos com IA, print simulator, RKW costing, Planjobs, previsto x realizado, financeiro e estoque. Fonte: https://holdprint.net/holdprint-desktop
- Prompt anexado: exige menu lateral vertical, topo compacto, Orçamento e O.S. sem wizard/dropdown/drawer, e uso do Caixa/PDV atual como referencia interna de layout.

## Melhor do Mubisys

1. Organizacao operacional por areas
   - Comercial
   - Orcamentos
   - O.S.
   - Producao / PCP
   - Caixa / PDV
   - Financeiro
   - Estoque
   - Relatorios
   - Configuracoes

2. Navegacao simples
   - Menu lateral vertical, uma opcao por linha.
   - Header fixo com busca, loja atual, usuario e indicadores rapidos.
   - Abas horizontais simples dentro de cada modulo.

3. Visual de ERP de rotina
   - Fundo claro.
   - Cards brancos.
   - Tabelas limpas.
   - Status coloridos.
   - Atalhos rapidos.
   - Tela pensada para operador, vendedor, caixa, producao e gestor.

4. Fluxo ponta a ponta
   - Lead/cliente.
   - Orcamento.
   - Aprovacao.
   - O.S.
   - Producao.
   - Entrega.
   - Financeiro.

## Melhor do Holdprint/Holdworks

1. Orcamento inteligente
   - Produto ou composicao define perguntas.
   - Questionario tecnico conforme tipo de servico.
   - Respostas alimentam materiais, horas, maquina, instalacao e margem.

2. Precificacao gerencial
   - Custo de material.
   - Custo de producao.
   - Custo de instalacao.
   - Despesas.
   - Ponto de equilibrio.
   - Margem minima.
   - Preco minimo.
   - Preco sugerido.
   - Preco final.
   - Alertas de venda abaixo do custo/margem.

3. Producao conectada ao orcamento
   - O que foi vendido vira fluxo de producao.
   - Materiais previstos entram no PCP/estoque.
   - Tempo previsto pode ser comparado ao tempo real.
   - Custos reais alimentam a rentabilidade.

4. Controle previsto x realizado
   - Preco calculado x preco aprovado.
   - Material previsto x usado.
   - Tempo previsto x real.
   - Margem prevista x margem real.
   - O.S. lucrativa ou com prejuizo.

## Melhor do prompt anexado

1. Caixa/PDV como padrao visual interno
   - Usar a tela de Caixa como referencia de organizacao.
   - Separar acoes por blocos claros.
   - Reduzir espacos vazios.
   - Manter operacao direta.

2. Menu lateral definitivo
   - Dashboard
   - Atendimento
   - Clientes
   - Orcamentos
   - Ordens de Servico
   - Producao / PCP
   - Caixa / PDV
   - Financeiro
   - Estoque
   - Relatorios
   - Configuracoes

3. Orçamento sem poluicao
   - Sem wizard.
   - Sem menu suspenso.
   - Sem menu flutuante.
   - Cabecalho, abas simples, cards, tabela, custos e questionario tecnico.

4. O.S. sem drawer lateral
   - Tudo dentro da pagina.
   - Tabela principal com ficha da O.S. abaixo ou ao lado.
   - Abas simples por status.
   - Acoes reais perto da O.S.

## Direcao ideal para o PrintSys

O PrintSys deve ter:

- Organizacao visual parecida com Mubisys.
- Inteligencia de orcamento parecida com Holdprint/Holdworks.
- Layout interno padronizado pelo Caixa/PDV atual.
- Identidade propria, com nome PrintSys e codigo proprio.

## Modelo de tela recomendado

### Estrutura global

1. Menu lateral vertical
2. Header compacto
3. Conteudo principal
4. Cabecalho do modulo
5. Filtros ou abas simples
6. Cards de resumo
7. Tabela ou formulario principal
8. Resumo operacional
9. Acoes no contexto certo

### Orçamento ideal

- Cabecalho: titulo, cliente atual, status, vendedor, prazo, total e acoes.
- Abas horizontais: Dados, Itens, Custos, Revisao.
- Cards: Cliente, Servico, Prazo, Total.
- Tabela: Item, Produto/Servico, Descricao, Medida, Qtd, Unitario, Total, Acoes.
- Questionario tecnico por composicao.
- Painel de custo com:
  - custo material;
  - custo producao;
  - instalacao;
  - despesas;
  - margem;
  - desconto;
  - preco minimo;
  - preco sugerido;
  - preco final.
- Alertas:
  - margem abaixo da meta;
  - margem critica;
  - venda abaixo do custo;
  - desconto exige aprovacao.

### O.S. ideal

- Cabecalho: numero, cliente, prazo, status producao, status financeiro, total e acoes.
- Abas: Todas, Hoje, Atrasadas, Em producao, Finalizadas, Sem arquivo, Sem pagamento.
- Cards: total, hoje, atrasadas, em producao, sem arquivo, sem pagamento.
- Tabela: O.S., Cliente, Servico, Prazo, Status producao, Status financeiro, Setor, Responsavel, Acoes.
- Ficha interna da O.S.:
  - cliente;
  - trabalho;
  - itens;
  - producao;
  - financeiro;
  - arquivos;
  - observacoes;
  - historico.

## O que evitar

- Copiar marca, logo, SVGs ou textos proprietarios.
- Menu superior como navegacao principal.
- Wizard em Orçamento/O.S.
- Dropdown para acoes principais.
- Drawer lateral na tela de O.S.
- Botao morto ou decorativo.
- Informacao tecnica exposta para usuario comum.
- Cards repetidos sem funcao.
- Tabelas gigantes sem hierarquia.

## Prompt consolidado para executar a proxima etapa

Repaginar o PrintSys usando Mubisys como referencia de organizacao visual e Holdprint/Holdworks como referencia de inteligencia de orcamento e producao.

Nao copiar marca, codigo, icones proprietarios, imagens ou textos.

Manter:

- autenticacao;
- permissoes;
- multiempresa;
- motor de precificacao;
- APIs existentes;
- persistencia;
- regras homologadas.

Prioridade:

1. Menu lateral vertical definitivo.
2. Header compacto com busca, loja, indicadores, usuario e sair.
3. Orçamento em tela profissional, sem wizard/dropdown/flutuante.
4. O.S. em tela profissional, sem drawer lateral.
5. PCP com tabela limpa, filtros e acoes reais.
6. Caixa/PDV mantendo padrao atual e melhorando separacao dos blocos.
7. Relatorios em cards simples.
8. Padronizacao global de cards, tabelas, botões, inputs, status e espacamento.

Critério de aceite:

- O usuário deve entender cada tela em menos de 5 segundos.
- Cada botao visivel deve executar acao real ou estar claramente desabilitado.
- Orcamento deve parecer inteligente, com custo, margem e alertas claros.
- O.S. deve parecer operacional, com status, setor, responsavel e acoes visiveis.
- Sistema deve parecer produto final, nao prototipo.
